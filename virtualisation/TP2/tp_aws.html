Note sur l'Environnement de Travail

Ce TP a été réalisé sur une instance AWS EC2. Le passage au Cloud a été motivé par des limitations techniques sur l'hôte physique, garantissant ainsi un environnement de conteneurisation stable et performant.

Partie 4 — Standards OCI

4.1L'initiative OCI

Garantie de l'interopérabilité entre les différentes solutions de conteneurs.

Réponse Question 11 : Les standards OCI permettent l'interopérabilité. Il est possible de créer un conteneur LXC à partir d'une image Docker en extrayant le rootfs et en générant un fichier config.json compatible.

4.2Inspection de l'image Alpine

docker save alpine -o alpine.tar tar xf alpine.tar && cat manifest.json

[Capture : Analyse du fichier manifest.json OCI]

Réponse Question 12 : Le sha256 assure l'intégrité de la donnée par identification de contenu. Cela garantit qu'aucune altération n'a eu lieu pendant le transfert entre le registre et l'instance AWS.

Conclusion du TP

La réalisation de ce TP sur AWS a permis de valider les compétences de déploiement et d'administration à distance via SSH, tout en approfondissant la maîtrise des mécanismes internes de Docker (Namespaces, Cgroups) et des standards OCI.
