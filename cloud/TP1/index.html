TP1 : Mise en service d'une instance EC2

Déploiement et configuration d'un serveur Ubuntu 22.04 LTS sur AWS.

1. Comprendre Amazon EC2

EC2 (Elastic Compute Cloud) est le service de calcul d'AWS. Il permet de louer des serveurs virtuels (instances) scalables.

  • Elastic : Capacité d'ajuster les ressources selon la charge.
  • Compute : Fournit la puissance de traitement nécessaire aux applications.

2. Déploiement de l'Infrastructure

1 Création du compte Free Tier

Accès aux services gratuits (750h/mois d'instance t2.micro).

2 Lancement de l'instance

Sélection de l'AMI Ubuntu Server 22.04 LTS et création de la paire de clés (Key Pair).

3 Security Groups (Pare-feu)

Configuration des règles entrantes (Inbound Rules) :

  • SSH (22) : Administration distante (limité à mon IP).
  • HTTP (80) : Accès public pour le serveur web.

3. Connexion au serveur

Avant la connexion, il est impératif de restreindre les droits sur la clé privée :

# Sur la machine locale (ThinkPad T14s) 
chmod 400 ma-cle-aws.pem
ssh -i "ma-cle-aws.pem" ubuntu@votre-ip-publique-aws

4. Commandes de configuration (Bash)

Une fois connecté, voici les étapes réalisées pour préparer le serveur :

# Mise à jour du système 
sudo apt update && sudo apt upgrade -y
# Installation du serveur Web Apache 
sudo apt install apache2 -y
sudo systemctl enable apache2
# Gestion des droits pour le dossier web 
sudo chown -R ubuntu:ubuntu /var/www/html
Résumé Technique
  • Région : us-east-1
  • Type : t2.micro (Free Tier)
  • vCPU : 1
  • RAM : 1 GiB
  • OS : Ubuntu 22.04
Statut : Opérationnel

Instance déployée avec succès et accessible via HTTP.
