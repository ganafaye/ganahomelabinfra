TP3 : Orchestration Multi-Services

Automatisation du déploiement d'une stack Web + Database avec Docker Compose.

1. Concept de l'Orchestration

Docker Compose permet de définir et de lancer des applications multi-conteneurs. Au lieu de gérer chaque conteneur manuellement sur l'instance EC2, on utilise un fichier déclaratif docker-compose.yml.

2. Configuration YAML

Ce fichier définit deux services : une application Python (Flask) et une base de données PostgreSQL.

version: '3.8'

services:
  db:
    image: postgres:15
    environment:
      POSTGRES_USER: gana
      POSTGRES_PASSWORD: mdp_secure
    volumes:
      - postgres_data:/var/lib/postgresql/data

  web:
    build: .
    ports:
      - "80:5000"
    depends_on:
      - db # L'app attend que la DB soit lancée

volumes:
  postgres_data: # Persistance des données

3. Mise en service sur AWS

Commandes exécutées sur le terminal de l'instance Ubuntu pour lancer la stack :

# Lancement en mode détaché 
sudo docker-compose up -d
# Vérification de l'état des services 
sudo docker-compose ps
Avantages clés
  • Isolation Réseau : Compose crée un réseau privé par défaut pour les services.
  • Persistance : Les volumes assurent que les données de la DB ne sont pas perdues au redémarrage.
  • Scalabilité : Possibilité de dupliquer les instances d'un service facilement.

L'application est désormais accessible sur le port 80 de l'IP publique AWS.
