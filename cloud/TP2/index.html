TP2 : Virtualisation & Déploiement Python

Conteneurisation d'une application de gestion des ressources sur Ubuntu EC2.

0. Connexion au Serveur AWS

Accès sécurisé depuis mon ThinkPad T14s vers l'instance Ubuntu.

chmod 400 key-aws.pem
ssh -i "key-aws.pem" ubuntu@votre-ip-aws

1. Installation de Docker Engine

Mise en place de l'environnement de virtualisation léger.

sudo apt update && sudo apt install docker.io -y
sudo systemctl start docker
sudo systemctl enable docker

2. L'Application de Gestion des Ressources

Création d'une mini-app app.py avec Flask pour afficher les informations système.

from flask import Flask
import os

app = Flask(__name__)

@app.route('/')
def home():
    return f"
Serveur de Gana Faye
Statut : Conteneur Isolé
"

if __name__ == "__main__":
    app.run(host='0.0.0.0', port=5000)

3. Isolation & Dockerfile

Définition de l'image pour isoler l'environnement Python.

FROM python:3.9-slim
WORKDIR /app
COPY . /app
RUN pip install flask
EXPOSE 5000
CMD ["python", "app.py"]

Construction et lancement :

sudo docker build -t mon-app-python .
sudo docker run -d -p 80:5000 --name gestion-ressources mon-app-python

4. Gestion des ressources & Tests

Docker permet de limiter la consommation de l'instance EC2 par le conteneur pour éviter les saturations :

# Lancement avec limitation (512 Mo RAM, 0.5 CPU)
sudo docker run -d --memory="512m" --cpus="0.5" -p 80:5000 mon-app-python

Vérification de l'isolation : Accédez à l'IP publique de votre instance sur votre navigateur pour voir l'application en direct.

Monitoring

Suivi des ressources en temps réel :

sudo docker stats
