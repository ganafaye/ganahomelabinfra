Note sur l'Environnement de Travail

Ce TP a été porté sur une instance AWS EC2. Le passage au Cloud permet d'expérimenter Docker dans un environnement de production réel, palliant les limitations techniques rencontrées sur l'hôte physique local.

Sécurisation de la clé privée .pem et établissement de la connexion cryptée.

# Sécurisation des permissions chmod 400 "gana-key.pem" # Connexion à l'instance distante ssh -i "gana-key.pem" ubuntu@ec2-XX-XX-XX-XX.compute-1.amazonaws.com

1Installation de Docker Engine

Mise en place du moteur de conteneurisation et vérification du runtime.

sudo apt-get update && sudo apt-get install -y docker-ce docker-ce-cli containerd.io sudo usermod -aG docker $USER && newgrp docker docker info

Réponse Question 5 : Le moteur utilise le runtime runc (conforme OCI) et le pilote overlay2 pour la gestion efficace des couches d'images.

Création d'un micro-service pour simuler une application de gestion de ressources.

from flask import Flask import os app = Flask(__name__) @app.route('/') def home(): return f"<h1>Serveur de Gana Faye</h1><p>Statut : Conteneur Isolé sur AWS</p>" if __name__ == "__main__": app.run(host='0.0.0.0', port=5000)

3Isolation par Dockerfile

Définition de l'environnement immuable pour l'application.

FROM python:3.9-slim WORKDIR /app COPY . /app RUN pip install flask EXPOSE 5000 CMD ["python", "app.py"]
# Build & Lancement docker build -t app-gana:v1 . docker run -d -p 80:5000 --name web-app app-gana:v1

[Capture : Application accessible via l'IP publique AWS]

Question 8 (Isolation) : En exécutant docker top, on observe que le processus Python possède un PID différent sur l'hôte et dans le conteneur, prouvant l'isolation par les Namespaces Linux.

Partie 4 — Standards OCI & Interopérabilité

4.1L'initiative OCI

Analyse de la structure standardisée des images pour garantir la portabilité entre registres et runtimes.

Réponse Question 11 : Les standards OCI (Open Container Initiative) permettent de découpler le format de l'image de l'outil d'exécution. Il est donc techniquement possible de convertir une image Docker en rootfs pour l'exécuter avec LXC.

4.2Inspection de l'image Alpine

docker save alpine -o alpine.tar tar xf alpine.tar && cat manifest.json

[Capture : Décomposition du manifest OCI de l'image Alpine]

Réponse Question 12 : L'utilisation du hash sha256 dans le manifest garantit l'intégrité technique. Lors du téléchargement, Docker compare le hash calculé au hash annoncé : s'ils diffèrent, l'image est rejetée pour prévenir toute corruption ou attaque.
