Ansible : Infrastructure as Code

Automatisation de la configuration et déploiement d'applications sans agent.

1. Définition

Ansible est un outil d'automatisation IT "agentless". Il utilise SSH pour se connecter aux machines cibles et exécuter des Playbooks écrits en YAML. Il est idéal pour configurer des serveurs de manière identique et reproductible.

2. Méthodes d'Installation

Ubuntu / Linux
sudo apt update
sudo apt install software-properties-common
sudo add-apt-repository --yes --update ppa:ansible/ansible
sudo apt install ansible -y
macOS
# Via Homebrew
brew install ansible
Windows

Note : Ansible ne tourne pas nativement sur Windows. Il faut utiliser WSL (Windows Subsystem for Linux).

# Dans le terminal WSL (Ubuntu)
sudo apt update && sudo apt install ansible -y

3. Utilisation : Le Playbook

Exemple d'un playbook pour installer Nginx sur un groupe de serveurs :

- name: Installation de Nginx
  hosts: webservers
  become: yes
  tasks:
    - name: Installer la dernière version de nginx
      apt:
        name: nginx
        state: latest
